ETL NY Taxi Data with Prefect

Project Goal

To build an efficient and reliable ETL pipeline for processing New York City taxi data, storing it in a local PostgreSQL database, and loading it into Google BigQuery for analysis. You can find the complete source code and documentation on GitHub: https://github.com/Ujeeg/ETL-NY-TAXI-DATA-With-Prefect

ETL Process

  1. **Extract:** Download and decompress NYC taxi data from a GitHub repository in CSV.GZIP format.
  2. **Transform:**
    • Convert date and time columns to the correct datetime format.
    • Clean and preprocess the data to handle missing values and ensure data quality.
  3. **Load:**
    • Load the transformed data into a local PostgreSQL database running in a Docker container.
    • Transfer the data from PostgreSQL to Google BigQuery for cloud-based analysis and reporting.

Challenges and Solutions

Due to limitations with my GCP account, I couldn't use a GCP bucket for staging the data before loading it into BigQuery. To overcome this, I implemented a direct load from the local PostgreSQL database to BigQuery using the BigQuery Python client library and secure authentication.

Results

This project resulted in a fully automated ETL pipeline orchestrated with Prefect. The pipeline effectively ingests, processes, and stores NYC taxi data, making it readily available for analysis in both local and cloud environments.
